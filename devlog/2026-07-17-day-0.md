## 2026-07-17 — Repo scaffold + architecture decisions

Set up the CMake/Cargo skeleton and wrote four ADRs: single-writer-per-symbol threading, array + intrusive-list order book replacing `std::map`, a `cxx`-for-control-plane / ring-buffer-for-data-plane FFI split, and an explicit v1 scope guard.

While wiring the build, found and fixed two real bugs in the original v1 code. `match()` took the taker order **by value**, so a fully-filled order's own quantity was never updated after matching — it got pushed back onto the book at its full original size instead of resting empty. None of the existing tests caught this since they only checked the returned trades, not book state after a fill; fixed by passing by reference and added a regression test that checks `depth()`/`best_bid()`/`best_ask()` after an exact full fill. Separately, `bids_` and `asks_` used different `std::map` comparator types, which broke a ternary-based book-selection expression — replaced with explicit branches.
